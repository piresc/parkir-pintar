#06
POST /api/v1/reservations/:id/complete

CompleteCheckout

Finalizes a checked-out reservation by generating an invoice, processing payment via QRIS, and releasing the parking spot back to available status.

Request Flow
Gateway
Extract reservation :id from path, call gRPC CompleteCheckout
Phase 1 — Transaction
GetByIDForUpdate → ownership check → status must be checked_out
Acquires row lock to prevent concurrent completion
Phase 2 — Billing
billingClient.GenerateInvoice (idempotent re-fetch)
If invoice already exists, fetches existing one instead of creating duplicate
Phase 3 — Payment
paymentClient.ProcessPayment(billingID, totalAmount, "qris", idempotencyKey)
QRIS payment processing with idempotency key for safe retries
Phase 4 — Finalize (Transaction)
Set status=completed, UpdateSpotStatusTx(available)
Spot released back to pool within same transaction
Events Published
Emit NATS events for search index and analytics
Events Emitted
reservation.search.spot-updated status → available
reservation.analytics.completed analytics tracking
SQL Queries
-- Phase 1: Lock reservation row
SELECT * FROM reservations
WHERE id = $1 FOR UPDATE;

-- Phase 4: Mark completed
UPDATE reservations
SET status = 'completed', completed_at = NOW()
WHERE id = $1;

-- Phase 4: Release spot
UPDATE parking_spots
SET status = 'available'
WHERE id = $1;
Response — 200 OK
{
  "id": "uuid",
  "status": "completed",
  "completed_at": "2026-05-20T10:30:00Z",
  "invoice": {
    "id": "inv-uuid",
    "total_amount": 15000
  },
  "payment": {
    "method": "qris",
    "status": "success"
  }
}
Error Responses
CodeConditionMessage

#07
DELETE /api/v1/reservations/:id

CancelReservation

Cancels a reservation that is in waiting_payment or confirmed status. No refund is issued — booking fee is forfeited if cancelled from confirmed state.

Request Flow
Gateway
Extract reservation :id from path, call gRPC CancelReservation
Transaction
GetByIDForUpdate → ownership check → ValidateTransition(→cancelled)
Only valid from: waiting_payment, confirmed
Cancel & Release
Set cancelled_at, UpdateSpotStatusTx(available)
Spot immediately returned to available pool
Events
Publish spot-updated + analytics cancelled events
NOTE: No refund is issued. If cancelled from confirmed status, the booking fee is forfeited as a cancellation penalty.
Events Emitted
reservation.search.spot-updated status → available
reservation.analytics.cancelled cancellation tracking
SQL Queries
-- Lock reservation
SELECT * FROM reservations
WHERE id = $1 FOR UPDATE;

-- Cancel reservation
UPDATE reservations
SET status = 'cancelled', cancelled_at = NOW()
WHERE id = $1;

-- Release spot
UPDATE parking_spots
SET status = 'available'
WHERE id = $1;
Response — 200 OK
{
  "id": "uuid",
  "status": "cancelled",
  "cancelled_at": "2026-05-20T10:30:00Z",
  "message": "reservation cancelled successfully"
}
Error Responses
CodeConditionMessage
404Reservation not foundreservation not found
403Not the ownerunauthorized access
409Invalid source statuscannot cancel from current state

GET /api/v1/reservations

ListByDriver

Lists all reservations for the authenticated driver. Driver ID is extracted from JWT (not query param). Optional ?status= filter.

Request Flow
Gateway
Extract driverID from JWT (not query param), read optional ?status=
Security: driver can only see their own reservations
Usecase
repo.ListByDriverID(driverID, status)
Joins with parking_spots for spot_code enrichment
Response
Return array of reservations ordered by created_at DESC
SQL Query
SELECT r.*, ps.spot_code
FROM reservations r
JOIN parking_spots ps ON ps.id = r.spot_id
WHERE r.driver_id = $1
  -- Optional status filter:
  [AND r.status = $2]
ORDER BY r.created_at DESC;

#10
GET /api/v1/availability

GetAvailability

Returns per-floor availability counts for a given vehicle type. Uses Redis cache with 5s TTL and singleflight to prevent thundering herd on cache miss.

Request Flow
Gateway
Read ?vehicle_type= query param, call gRPC GetAvailability
gRPC Handler
Validate vehicle_type is required, compute totals from floor data
Cache Check
redis.Get("availability:<vehicleType>") — 5s TTL
If hit → return cached result immediately
Cache Miss
singleflight → repo.GetAvailabilityByVehicleType → cache result
Singleflight deduplicates concurrent requests for same key
Response
Return per-floor availability with totals
Caching Strategy
// Redis cache key pattern
Key:  "availability:<vehicleType>"   // e.g. "availability:car"
TTL:  5 seconds
Invalidation: NATS consumer on spot-updated events
SQL Query (on cache miss)
SELECT
    floor_number,
    COUNT(*) FILTER (WHERE status = 'available' AND vehicle_type = $1) AS available,
    COUNT(*) FILTER (WHERE vehicle_type = $1) AS total
FROM spot_read_model
GROUP BY floor_number;
Response — 200 OK
{
  "vehicle_type": "car",
  "total_available": 42,
  "total_capacity": 120,
  "floors": [
    { "floor": 1, "available": 8, "total": 24 },
    { "floor": 2, "available": 12, "total": 24 },
    { "floor": 3, "available": 10, "total": 24 },
    { "floor": 4, "available": 6, "total": 24 },
    { "floor": 5, "available": 6, "total": 24 }
  ]
}

POST /api/v1/presence/stream

StreamPresence

Verifies driver physical presence at their reserved parking spot using sensor data. Gracefully degrades if sensor gateway is unavailable — assumes presence.

Request Flow
Gateway
Parse JSON body, extract driverID from JWT
Body: {reservation_id, latitude, longitude, accuracy}
gRPC Handler
Validate driver_id + reservation_id combination
Sensor Check
sensor.CheckSpotOccupancy(floorNumber, spotNumber)
External call to sensor gateway (currently stub)
Result Mapping
Map sensor response to verification result
Request Body
{
  "reservation_id": "reservation-uuid",
  "latitude": -6.2088,
  "longitude": 106.8456,
  "accuracy": 5.2
}
Sensor Response Matrix
Sensor ResultverifiedMessage
Occupied true presence confirmed
Not occupied false driver may be at wrong spot
Sensor unavailable true sensor unavailable, presence assumed
No DB queries. This endpoint only communicates with the external sensor gateway. Currently stubbed — always returns "occupied" in development.
Response — 200 OK
// Presence confirmed
{
  "verified": true,
  "message": "presence confirmed"
}

// Sensor unavailable (graceful degradation)
{
  "verified": true,
  "message": "sensor unavailable, presence assumed"
}

// Not at spot
{
  "verified": false,
  "message": "driver may be at wrong spot"
}
Error Responses
CodeConditionMessage
400Missing required fieldsinvalid request body
403Driver doesn't own reservationunauthorized access
NATS Event Consumers

Background flows that react to domain events. These consumers run as long-lived goroutines subscribing to NATS JetStream subjects.

Search Consumer
Subject: reservation.search.spot-updated
Receive Event
Unmarshal SpotUpdatedEvent from NATS message
Contains: spot_id, new_status, vehicle_type, floor_number
Update Read Model
Upsert spot_read_model with new status
Invalidate Cache
Delete Redis keys: availability:*, floormap:*
Forces next read to fetch fresh data from DB
-- Upsert spot read model
INSERT INTO spot_read_model (id, spot_code, vehicle_type, status, floor_number, spot_number)
VALUES ($1, $2, $3, $4, $5, $6)
ON CONFLICT (id) DO UPDATE SET
    status = EXCLUDED.status;
Reservation Consumer
Subjects: payment.reservation.success / payment.reservation.failed
Receive Event
Unmarshal PaymentResultEvent from NATS message
Contains: reservation_id, payment_status, payment_id
Success Path
ConfirmReservation — set status=confirmed, confirmed_at=NOW()
Failure Path
FailReservation — set status=payment_failed, release spot
-- On payment success
UPDATE reservations
SET status = 'confirmed', confirmed_at = NOW()
WHERE id = $1;

-- On payment failure
UPDATE reservations
SET status = 'payment_failed'
WHERE id = $1;

UPDATE parking_spots
SET status = 'available'
WHERE id = $1;
Analytics Consumer
Subject: reservation.analytics.*
Receive Event
Unmarshal ReservationEvent from NATS message
Wildcard subscription catches: .created, .confirmed, .checked_in, .completed, .cancelled
Record
RecordEvent — INSERT into analytics table
Fire-and-forget: failures logged but don't block
-- Record analytics event
INSERT INTO reservation_analytics
    (reservation_id, event_type, vehicle_type, floor_number, spot_id, occurred_at)
VALUES
    ($1, $2, $3, $4, $5, $6);
