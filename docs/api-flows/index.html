#02 POST /api/v1/reservations CreateReservation

Creates a new parking reservation with full idempotency support, distributed locking, transactional spot assignment, billing integration, and async task scheduling.

High-Level Flow
Gateway
Parse JSON, enforce ownership (driver_id == JWT user), build gRPC request
gRPC Handler
Validate required fields, delegate to usecase
Usecase
Idempotency → Lock → Transaction → Billing → Tasks → Events
Repository
PostgreSQL queries within transaction
Usecase Detail Steps
Step 1 — Idempotency Check
Call repo.FindByIdempotencyKey(key). If found, return existing reservation immediately (short-circuit).
Step 2 — Spot Assignment
If system_assigned: call FindAvailableSpot(vehicleType) to get next available spot.
If user_selected: use the provided req.SpotID directly.
Step 3 — Distributed Lock
Acquire Redis lock via SETNX on key "spot:<spotID>" with TTL. Prevents concurrent reservations on same spot.
Step 4 — Transaction
Within a single DB transaction:
• GetSpotForUpdateTx — row-level lock on spot
• Verify spot status == available
• CreateReservationTx — insert reservation row
• UpdateSpotStatusTx — set spot to reserved
Step 5 — Billing
Call billingClient.StartBilling(reservationID, 5000, idempotencyKey) to initialize billing session.
Step 6 — Async Tasks
• EnqueueReservationExpiry — fires after 60 minutes
• EnqueuePaymentHoldTimeout — fires after 10 minutes
Step 7 — Events
Publish NATS events for downstream consumers.

#03 POST /api/v1/reservations/:id/confirm ConfirmReservation

Confirms a reservation by processing payment. Uses a multi-phase approach with TOCTOU protection to ensure consistency between payment processing and status updates.

High-Level Flow
Gateway
Extract :id param, call gRPC ConfirmReservation
gRPC Handler
Validate reservation_id, extract CallerID from metadata
Usecase
4-phase: Lock → Billing → Payment → Commit
External
Billing + Payment services
Multi-Phase Execution
Phase 1 — Initial Lock & Validation (Transaction)
GetByIDForUpdate — acquire row lock on reservation.
Verify caller == owner (ownership check).
Verify status == waiting_payment.
Phase 2 — Billing (Idempotent)
Call billingClient.StartBilling(reservationID, 5000, idempotencyKey).
This is a re-call (idempotent) — billing was already started at creation. Returns existing billing session ID.
Phase 3 — Payment Processing
Call paymentClient.ProcessPayment(billingID, 5000, "qris", idempotencyKey).
Processes actual payment via QRIS method. Returns payment confirmation or error.
Phase 4 — Final Commit (Transaction + TOCTOU Guard)
Re-fetch reservation with FOR UPDATE lock.
Verify status is STILL waiting_payment (TOCTOU protection against concurrent changes).
Set status = confirmed, confirmed_at = now, expires_at = now + TTL.
Post-Commit — Side Effects
Publish reservation.analytics.confirmed event.
Cancel the payment-hold-timeout async task (no longer needed).
Published Events (NATS)
reservation.analytics.confirmed
External Service Calls
// Phase 2 — Billing (idempotent re-call) billingClient.StartBilling(reservationID, 5000, idempotencyKey) // Phase 3 — Payment paymentClient.ProcessPayment(billingID, 5000, "qris", idempotencyKey)
Error Cases
StatusConditionResponse
404Reservation not foundReservation not found
403Caller is not the reservation ownerNot authorized to confirm this reservation
400Status is not waiting_paymentReservation cannot be confirmed in current state
402Payment processing failedPayment failed: {reason}
409Concurrent status change detected (TOCTOU)Reservation was modified concurrently, retry
#04 POST /api/v1/reservations/:id/checkin CheckIn

Records driver arrival at the parking spot. Transitions reservation to checked_in, marks spot as occupied, activates billing timer, and optionally verifies physical presence.

High-Level Flow
Gateway
Extract :id, call gRPC CheckIn
gRPC Handler
Validate ID, extract CallerID
Usecase
Transaction → Billing → Presence → Events
Repository
Update reservation + spot status
Usecase Detail Steps
Step 1 — Transaction
GetByIDForUpdate — lock reservation row.
Ownership check (caller == driver_id).
ValidateTransition(→checked_in) — current status must be confirmed.
Set checked_in_at = now.
UpdateSpotStatusTx(occupied) — mark parking spot as occupied.
Step 2 — Billing Activation
Call billingClient.StartBilling(reservationID).
This activates the per-minute billing timer for the parking session.
Step 3 — Presence Verification (Non-Blocking)
Call presenceClient.VerifyPresence(reservationID).
Graceful degradation — if presence service is unavailable, check-in still succeeds. Result is logged but does not block the flow.
Step 4 — Events
Publish events for search index update and analytics tracking.
Published Events (NATS)
reservation.search.spot-updated reservation.analytics.checked-in
External Service Calls
// Activate billing timer billingClient.StartBilling(reservationID) // Non-blocking presence check (graceful degradation) presenceClient.VerifyPresence(reservationID)
Note: The spot status changes from reserved → occupied at check-in. This is published via the reservation.search.spot-updated event to keep the search index current.
#05 POST /api/v1/reservations/:id/checkout CheckOut

Initiates the checkout process. Calculates parking fees based on duration, generates an invoice, and returns a billing breakdown. Note: the spot remains "occupied" until a separate CompleteCheckout flow releases it.

High-Level Flow
Gateway
Extract :id, call gRPC CheckOut
gRPC Handler
Validate ID, extract CallerID
Usecase
Transaction → Fee Calculation → Invoice
Billing
CalculateFee + GenerateInvoice
Usecase Detail Steps
Step 1 — Transaction
GetByIDForUpdate — lock reservation row.
Ownership check (caller == driver_id).
ValidateTransition(→checked_out) — current status must be checked_in.
Set checked_out_at = now.
Step 2 — Fee Calculation
Call billingClient.CalculateFee(reservationID, checkedInAt, checkedOutAt).
Computes total parking fee based on duration and rate tiers.
Step 3 — Invoice Generation
Call billingClient.GenerateInvoice(reservationID).
Creates a formal invoice record with line items and total.
Step 4 — Response
Return CheckOutResponse with full billing breakdown.
No payment is collected at this stage — that happens in a separate payment flow.
Important: The parking spot remains in occupied status after checkout. It is only released back to available when the CompleteCheckout flow runs (after payment is settled). This prevents the spot from being re-assigned before the vehicle physically leaves.
External Service Calls
// Calculate parking fee based on duration billingClient.CalculateFee(reservationID, checkedInAt, checkedOutAt) // Generate invoice with line items billingClient.GenerateInvoice(reservationID)
Success Response — 200
JSON { "reservation_id": "res-uuid-5678", "status": "checked_out", "checked_in_at": "2025-01-15T08:00:00Z", "checked_out_at": "2025-01-15T11:30:00Z", "billing": { "duration_minutes": 210, "base_rate": 5000, "total_fee": 17500, "invoice_id": "inv-uuid-9012" } }
#06 POST /api/v1/reservations/:id/complete CompleteCheckout

Finalizes a checked-out reservation by generating an invoice, processing payment via QRIS, and releasing the parking spot back to available status.

Request Flow
Gateway
Extract reservation :id from path, call gRPC CompleteCheckout
Phase 1 — Transaction
GetByIDForUpdate → ownership check → status must be checked_out. Acquires row lock to prevent concurrent completion.
Phase 2 — Billing
billingClient.GenerateInvoice (idempotent re-fetch). If invoice already exists, fetches existing one instead of creating duplicate.
Phase 3 — Payment
paymentClient.ProcessPayment(billingID, totalAmount, "qris", idempotencyKey). QRIS payment processing with idempotency key for safe retries.
Phase 4 — Finalize (Transaction)
Set status=completed, UpdateSpotStatusTx(available). Spot released back to pool within same transaction.
Events Published
Emit NATS events for search index and analytics.

#10 GET /api/v1/availability GetAvailability

Read ?vehicle_type= query param, call gRPC GetAvailability
gRPC Handler
Validate vehicle_type is required, compute totals from floor data
Cache Check
redis.Get("availability:<vehicleType>") — 5s TTL. If hit → return cached result immediately.
Cache Miss
