SERVICES Microservices Architecture

  • Gateway — HTTP :8082 | Gin | REST API, JWT auth, rate limiting, CORS
  • Reservation — gRPC :9091 | Core domain, state machine, orchestrates billing & payment
  • Search — gRPC :9092 | CQRS read model, spot availability, synced via NATS
  • Billing — gRPC :9093 | Fee calculation, invoice generation, pricing rules
  • Payment — gRPC :9094 | QRIS/card/e-wallet, stub gateway, publishes results

EVENTS Event-Driven Architecture

  • RESERVATION_SEARCH — InterestPolicy | reservation.search.spot-updated
  • RESERVATION_ANALYTICS — LimitsPolicy 7d | reservation.analytics.* (created/confirmed/checked-in/completed/cancelled/expired)
  • PAYMENT_RESERVATION — InterestPolicy | payment.reservation.success/failed
waiting_payment→ confirmed→ checked_in→ checked_out→ completed

OBSERVABILITY Monitoring Stack

  • Alloy :4319 — OTLP collector, receives traces/metrics/logs from all services
  • Tempo :3200 — Distributed tracing backend (span metrics → Prometheus)
  • Prometheus :9090 — Metrics store, 30d retention, scrapes exporters
  • Loki :3100 — Log aggregation, 7d retention
  • Grafana :3000 — 3 dashboards: Executive, Infrastructure, Microservices
  • Pipeline: Services → OTLP → Alloy → Tempo + Prometheus + Loki → Grafana
