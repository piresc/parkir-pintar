Manage Data in Application

Sub: CRUD operations, transaction/ACID, locking, concurrency, distributed data

Level 4
🔐

Three-Layer Double-Booking Prevention

Layer 1: Redis distributed lock per spot (cross-process mutual exclusion). Layer 2: PostgreSQL SELECT FOR UPDATE (row-level lock within transaction). Layer 3: Partial unique index on active reservations per spot (idempotency + constraint). Defense-in-depth prevents TOCTOU races.

💱

Transactional Integrity (ACID)

WithTransaction pattern for atomic multi-table operations. Used in reservation lifecycle: create reservation + update spot status in one transaction. Optimistic concurrency via version column on payments (UpdatePaymentWithStatusCheck prevents double-refund).

🔄

Event-Driven Data Flow (NATS JetStream)

Async events decouple services: payment confirmed → reservation updated → billing calculated → search index refreshed. At-least-once delivery with explicit ack/nak. Message deduplication. Consumer replay for catch-up after downtime.

🗄️

Multiple Storage Paradigms

PostgreSQL (relational ACID), Redis (key-value state + distributed locks + rate limiting + idempotency cache), NATS JetStream (durable event streams). sqlx with parameterized queries throughout — zero string interpolation in SQL. TracedPostgresClient wraps queries with OpenTelemetry spans.

Rate Limiting — Both Transport Layers

Token bucket algorithm (100 req/s, burst 200) per IP. Applied at HTTP (Gin middleware) and gRPC (unary interceptor). Thread-safe with mutex. Automatic cleanup of stale entries every 5 minutes. Circuit breaker (sony/gobreaker) on external service calls.

⚠️

Failure Handling — Retry, Circuit Breaker, Fallback

gRPC client with configurable retry policy. Per-RPC deadlines. Circuit breaker wraps payment/billing clients — opens on repeated failures, half-open probing. Graceful degradation: payment failure doesn't crash reservation; non-core services (analytics) silently degrade.

JWT Authentication — Algorithm Pinning

HS256 with jwt.WithValidMethods([]string{"HS256"}) — prevents algorithm confusion. 32-char minimum secret in non-local environments. Claims: UserID, Role, ExpiresAt, IssuedAt, Issuer. Dual middleware: HTTP (Gin) + gRPC (interceptor). BYO-JWT for super-app integration.

Defense-in-Depth Authorization

Ownership enforced at gateway (cancels own reservations only, cannot create for other drivers) AND usecase layer (GetReservation checks callerID). Fail-closed: uid=="" returns 401. Admin-only endpoints with requireAdmin() middleware. gRPC errors map to PermissionDenied codes.
